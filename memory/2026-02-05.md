### 2. Long-Running Daemon Pattern Established
**Principle:** Spawn ONCE, reuse continuously (vs spawn-per-task)

**Conversions:**
- **Polling Reporter:** 0 spawns (pure bash daemon)
- **Background Indexer:** Writes to shared inbox (0 spawns)
- **MemoryJudge:** 1 spawn total, processes everything

**Spawn Reduction:** ~300/day → ~1 total

## Lessons Learned

1. **Simplicity beats complexity** - 8 layers → 1 daemon, works better
2. **Spawn once, reuse** - Pattern applies to any continuous task
3. **Character count ≠ meaning** - "You're amazing!" is important
4. **Check API keys** - kimi-code ≠ moonshot, different credentials
5. **Tyson was right** - Every time I over-engineered, simpler solution worked
